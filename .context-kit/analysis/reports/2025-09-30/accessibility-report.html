Executive Summary

The tkr-portfolio project demonstrates good baseline accessibility practices with several areas requiring improvement to achieve full WCAG 2.1 Level AA compliance. The codebase shows strong implementation of semantic HTML, ARIA attributes, and keyboard navigation in many components. However, critical issues were identified in heading hierarchy, form labeling, color contrast ratios, and alternative text for images.

Strengths

  • Excellent use of semantic landmarks (nav, main, footer, header)
  • Consistent ARIA labeling on interactive components
  • Keyboard navigation support with arrow keys in carousel
  • Screen reader text using sr-only classes
  • Focus management in modals and dropdowns
  • Progressive enhancement with proper fallbacks

Areas Requiring Attention

  • Inconsistent heading hierarchy (skipping levels)
  • Missing alt text on decorative and functional images
  • Color contrast issues on project picker buttons
  • Incomplete form labeling and error associations
  • Some interactive elements lack visible focus indicators
  • Missing skip navigation links

Critical Violations (Level A/AA)

CRITICAL
Missing Alternative Text on Images WCAG 1.1.1 (A)
Multiple images throughout the application lack proper alt text attributes or have generic alt text like "Image slide X". This prevents screen reader users from understanding the content and purpose of images.
User Impact:
Screen reader users cannot understand visual content. Decorative images should have empty alt="" attributes, while functional images need descriptive text.
Current Implementation:
// image-carousel.jsx (line 172-175) <img src={`${import.meta.env.BASE_URL}${item.src}`} alt={item.alt || `Image slide ${index + 1}`} // Generic fallback className="w-full h-full object-cover" />
Recommended Fix
Ensure all images have meaningful alt text passed from project data. For decorative images in slides, use alt="". For functional images, describe the content and purpose.
// Proper implementation <img src={src} alt={item.isDecorative ? "" : item.altText} className="w-full h-full object-cover" />

Inconsistent Heading Hierarchy WCAG 1.3.1 (A)
The application lacks a consistent heading structure. Many components jump from h2 to h4, skipping h3. The main App component uses a <main> landmark but no page-level h1 heading on most pages.
User Impact:
Screen reader users rely on heading hierarchy for page navigation. Skipping levels creates confusion about content organization and makes it difficult to understand page structure.
Violation Examples:
// AIProgressAssessment.jsx - skips h3 <h2>Calculating Your Results...</h2> ... <h4 className="font-semibold mb-3">{question.text}</h4> // AgentConversationFlow.jsx - inconsistent levels <h3>...</h3> <h2>...</h2> // h2 appears after h3
Recommended Fix
  1. Add an h1 element to each page/route (HomePage, ProjectPage, DemoShowcase)
  2. Ensure heading levels are sequential (h1 ‚Üí h2 ‚Üí h3, never skip levels)
  3. Review all slide components and adjust heading hierarchy

Insufficient Color Contrast WCAG 1.4.3 (AA)
The primary brand color (#613CB0) on white backgrounds provides insufficient contrast in several components. Text on colored project cards may not meet the 4.5:1 ratio for normal text or 3:1 for large text.
User Impact:
Users with low vision or color blindness cannot read text with insufficient contrast. This affects readability and comprehension for approximately 8% of male users and 0.5% of female users.
Contrast Calculations:

Recommended Fix

  2. Increase bg-black opacity from 40% to 60% for button overlays

Missing Skip Navigation Link WCAG 2.4.1 (A)
The application does not provide a "skip to main content" link at the beginning of the page. Keyboard users must tab through all navigation links before reaching the main content.
User Impact:
Keyboard-only users waste time and effort tabbing through repetitive navigation on every page load. This is especially frustrating for users with motor disabilities.
Recommended Fix
Add a skip link as the first focusable element in App.jsx:
// App.jsx - add before Navigation <a href="#main-content" className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:bg-primary focus:text-white focus:px-4 focus:py-2 focus:rounded" > Skip to main content </a> <main id="main-content" className="flex-grow...">

Carousel Indicators Lack Accessible Names WCAG 4.1.2 (A)
The carousel dot indicators have aria-label but do not announce which slide is currently active to screen reader users. The active state is only communicated visually through opacity.
User Impact:
Screen reader users cannot determine which slide is currently visible without navigating to the slide content itself.
Current Implementation:
// image-carousel.jsx (line 190-201) <button key={index} className={cn( "w-2 h-2 rounded-full transition-opacity", index === current ? "opacity-100" : "opacity-50" )} onClick={() => api?.scrollTo(index)} aria-label={`Go to slide ${index + 1}`} />
Recommended Fix
Add aria-current="true" to the active indicator and improve labels:
<button key={index} className={...} onClick={() => api?.scrollTo(index)} aria-label={`${index === current ? 'Current slide' : 'Go to slide'} ${index + 1} of ${items.length}`} aria-current={index === current ? "true" : undefined} />
